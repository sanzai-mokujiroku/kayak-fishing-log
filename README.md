# 釣行記録（カヤックフィッシング用ログ / PWA）

カヤックフィッシングの「1回の釣行」と、その中の「複数の釣果」を記録する個人用ツール。
スマホのホーム画面から起動でき、電波の無い海の上でもオフラインで使える。
記録は**その端末のブラウザ内（localStorage）だけ**に保存され、サーバーには一切送られない。

## できること
- 釣行ごとの共通条件（日付・場所・潮回り・潮汐メモ・天気・風・波高・水温）
- 釣果を匹ごとに追加（ヒット時刻・ジグ・アクション・反応層・魚種・サイズ・メモ）
- ジグ在庫の管理、魚種リストの追加
- Markdownコピー（1件／全件）、CSV書き出し（Excel対応・BOM付き）
- **バックアップ書き出し／取り込み**（JSON。機種変更やデータ消去に備える命綱）
- 満足度、釣行一覧の折りたたみ表示

## ファイル構成
公開に必要なのは以下だけ（`src/` はソース、`node_modules/` は開発用）。
```
index.html            画面の入れ物
app.js                アプリ本体（src/main.jsx をビルドしたもの・React同梱）
manifest.webmanifest  ホーム画面追加用の設定
sw.js                 オフライン用サービスワーカー
icon-180/192/512.png  アイコン
src/main.jsx          ソース（編集はここ）
make_icon.py          アイコン生成スクリプト
```

## 編集して作り直す手順
`src/main.jsx` を直したら、ビルドして `app.js` を作り直す：
```bash
npm install      # 最初の1回だけ
npm run build    # src/main.jsx → app.js
```
アイコンを作り直すとき：`npm run icons`
アプリを更新したら `sw.js` の `CACHE = "fishing-log-v1"` の番号を上げる（古いキャッシュを捨てて新版を配るため）。

## 公開（GitHub Pages）
1. 新しいリポジトリ（例: `kayak-fishing-log`）を作って、このフォルダの中身を push
2. Settings → Pages で公開ブランチを指定
3. 公開URLをスマホのブラウザで開き、Safariの共有ボタン →「ホーム画面に追加」

> PWA（ホーム画面追加・オフライン）はHTTPSが必要。GitHub PagesはHTTPSなのでそのまま使える。

## ローカルで動作確認
```bash
python3 serve.py   # http://127.0.0.1:4173
```

## 注意
- ブラウザのデータ消去・機種変更で記録は消える。定期的にバックアップ書き出しを。
- コピー・CSVには場所（緯度経度）がそのまま入る。ブログ・SNSに載せる時は必ず削除・ぼかす。
